# Modeller

This directory contains the core implementation of the JA4+ traffic classification prototype, along with evaluation and analysis scripts used to measure its performance.

## Directory Structure

```text
Modeller/
├── analysis/       # Scripts for measuring latency, accuracy, and visualizing results
├── data/           # Stored machine learning models and dataset references
├── evaluations/    # Evaluation scripts for different classification approaches
├── prototype/      # The classification pipeline and GUI application
└── results/        # Output directory for generated graphs and metrics
```

---

### 1. `prototype/`
Contains the final implementation of the hybrid classification prototype.
*   **`app_gui.py`**: A Graphical User Interface (GUI) for inputting JA4/JA4S/JA4TS fingerprints and viewing classification results in real-time.
*   **`pipeline_model/`**: The core library powering the classification logic.
    *   **`pipeline.py`**: The primary orchestrator managing the two-stage classification process (Database Lookup -> Random Forest).
    *   **`database_lookup.py`**: Handles fast, deterministic hierarchical lookups for known fingerprints.
    *   **`models.py`**: Defines and loads the machine learning models (e.g., Random Forest) used when a database lookup yields partial or no results.
    *   **`ja4_parser.py`**: Parses JA4/JA4S strings and extracts/parameterizes features for the machine learning models.

### 2. `evaluations/`
A suite of scripts designed to benchmark different classification strategies.
*   **`eval_pipeline.py`**: Evaluates the full hybrid system (Database + ML).
*   **`eval_random_forest.py`**: Benchmarks the Random Forest model in isolation.
*   **`eval_foxio.py`**: Benchmarks the FoxIO dictionary approach, specifically measuring collision rates and ambiguity per fingerprint.
*   **`eval_egenlagd.py`**: Evaluates a custom baseline model ("egenlagd").
*   **`compare_all.py`**: An automated runner that executes multiple evaluation scripts sequentially to generate comparative metrics.

### 3. `analysis/`
Utility scripts for interpreting data output from the evaluation suite.
*   **`visualize_results.py`**: Generates charts and graphs (e.g., accuracy distributions, source breakdowns) saved to the `results/` folder.
*   **`analyze_cm.py`**: Parses Confusion Matrices to identify specific misclassification patterns.
*   **`measure_accuracy.py`**: Calculates precision and accuracy percentages for the ML models on training and test datasets.
*   **`measure_latency.py`**: Benchmarks the execution time of the classification pipeline (average, median, min, max) to ensure real-time viability.

### 4. `results/`
Contains the visual and text outputs generated by the analysis tools, including graphs like `app_distribution.png`, `confusion_matrices_dual.png`, and summaries like `thesis_results.md`.

### 5. `data/`
Contains the serialized machine learning models (e.g., `.pkl` files) loaded by the classification pipeline.
